3. DATA FLOW EXAMPLE: User Clicks "Merge Sort"
═══════════════════════════════════════════════════════════════════

STEP 1: On Homepage
┌──────────────────────────────────┐
│  <div class="algorithm-card"     │
│    onclick="goTo('merge-sort')"> │  ← User clicks this
│    <h3>Merge Sort</h3>           │
│  </div>                          │
└──────────────────────────────────┘
              ↓
              
STEP 2: JavaScript Executes
┌──────────────────────────────────────────┐
│  function goTo(algoKey) {                │
│    window.location.href =                │
│      `visualizer.html?algo=${algoKey}`;  │
│  }                                       │
└──────────────────────────────────────────┘
              ↓

STEP 3: Browser Navigates
┌──────────────────────────────────────────┐
│  URL becomes:                            │
│  visualizer.html?algo=merge-sort         │
│                   ^^^^^^^^^^^^           │
│                   This is the key!       │
└──────────────────────────────────────────┘
              ↓

STEP 4: Visualizer Reads URL
┌──────────────────────────────────────────┐
│  const params = new URLSearchParams(     │
│    window.location.search                │
│  );                                      │
│  const algoKey = params.get('algo');     │
│  // algoKey = "merge-sort"               │
└──────────────────────────────────────────┘
              ↓

STEP 5: Looks Up Data
┌──────────────────────────────────────────┐
│  const algoData = DSAData[algoKey];      │
│  // Gets all merge sort info             │
│  // {                                    │
│  //   name: "Merge Sort",                │
│  //   complexity: "O(n log n)",          │
│  //   ...                                │
│  // }                                    │
└──────────────────────────────────────────┘
              ↓

STEP 6: Displays Everything
┌──────────────────────────────────────────┐
│  document.getElementById('title')        │
│    .textContent = algoData.name;         │
│                                          │
│  document.getElementById('complexity')   │
│    .textContent = algoData.timeComplexity│
│  // etc.                                 │
└──────────────────────────────────────────┘

5. WHY THIS ARCHITECTURE IS BRILLIANT
═══════════════════════════════════════════════════════════════════

❌ BAD APPROACH (Don't do this):
   60 separate HTML files:
   - binary-search.html
   - merge-sort.html
   - quick-sort.html
   - ... (57 more files)
   
   Problems:
   • Change design → Edit 60 files
   • Fix bug → Edit 60 files  
   • Add feature → Edit 60 files
   • Nightmare to maintain!

✅ GOOD APPROACH (What we built):
   1 visualizer.html + data-driven
   
   Benefits:
   • Change design → Edit 1 file
   • Fix bug → Edit 1 file
   • Add algorithm → Edit data file only
   • Professional & scalable!

6. REAL-WORLD ANALOGY
═══════════════════════════════════════════════════════════════════

Think of it like a restaurant:

❌ BAD: Having 60 different restaurants (one for each dish)
   → Want to change decor? Change 60 restaurants!
   
✅ GOOD: One restaurant with a menu (data file)
   → Want to change decor? Change one restaurant!
   → Want new dish? Just add to menu!

Our system:
  visualizer.html = The restaurant (same layout)
  algorithms-data.js = The menu (list of items)
  URL parameter = The order ticket
